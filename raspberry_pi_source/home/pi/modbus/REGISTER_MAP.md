### ⚠️ 이전 매핑과의 차이 (2026-04-07 수정)

기존 REGISTER_MAP(3/17)에서는 +2/+4를 값A/값B로 기재했으나,
2026-04-07 raw 레지스터 전수 검증 결과 **실제 사료량은 +3(Silo1), +4(Silo2)** 에 저장됨.

```diff
- +2: 빈1 값A → SCADA 총량 태그
- +4: 빈1 값B → 실제 사료 재고량
+ +2: Silo2 존재 여부 (0/1)
+ +3: Silo1 Weight (사료 재고량) ← 핵심
+ +4: Silo2 Weight (사료 재고량)
+ +5: Silo1 Feeding (급이량)
+ +6: Silo2 Feeding (급이량)
```

## 데이터 형식

- 모든 값은 **uint16** (부호 없는 16비트 정수)
- 단위: kg
- 범위: 0 ~ 약 11,000
